Rule A — Status always carries its code, on the left

Rule B — Activity ID + title inline; title wraps, chip anchors

✅ Do

Strengthening Maternal & Newborn Health Services in Conflict-Affected Areas (SMNHS)

Chip rendered inside the heading with mr-1.5 align-middle whitespace-nowrap; heading is font-medium leading-tight min-w-0 [text-wrap:wrap] so the title wraps under itself while the chip holds the first line.
❌ Don't
PARTNER-41119000
Strengthening Maternal & Newborn Health Services in Conflict-Affected Areas
Never stack the ID above/below the title in lists, and never truncate the title to keep one line — wrap it.

Rule C — Acronym matches the title exactly

✅ Do
Central Dry Zone Smallholder Agriculture and Climate Resilience Project (AGRI-CDZ)
Single space + parentheses; the acronym span inherits the title's family, size, and weight (font-medium text-foreground).
❌ Don't
Central Dry Zone Smallholder Agriculture and Climate Resilience Project AGRI-CDZ
No smaller/lighter/muted acronyms, no separators other than parentheses, no acronym-as-chip.

Rule D — Numbers right, currency prefix muted

✅ Do
USD36.0m
Numeric cells are text-right tabular-nums; "USD" is text-helper text-muted-foreground, the value is foreground 500.
❌ Don't
$36,000,000.00 USD
No left-aligned numbers, no symbol-prefix in tables, no equal-emphasis currency code.

Rule E — Tables are square; modal headers are surface-muted

Table wrappers never get rounded-*. Every dialog header is shaded bg-surface-muted (#f6f5f3) with a bottom border, and dialog titles are text-only (no leading icon).

8 · Form elements

All field-shaped controls share one silhouette: 40px tall, 6px radius, border-input hairline, white bg, 16px text. Label above at 16px medium (ui/label.tsx); helper below at 14px muted. Focus = 2px ring with 2px offset, app-wide.

Text input & textarea

Helper text sits below at 14px muted.

Textarea: min-h-[80px] resize-y, otherwise identical to the input recipe.

Select & date-picker triggers

Same recipe as the input (h-10 rounded-md border-input bg-background px-3 py-2 text-body) with justify-between and a muted chevron/calendar adornment. Unselected value = text-muted-foreground. Calendar selected day = pill on #1d1f21.

Checkbox & switch

Unchecked ✓ Checked Off On

Checkbox: h-4 w-4 rounded-sm border-primary, checked fills bg-primary. Switch is the one control that uses brand colors: checked #4c5568 (blue-slate), unchecked #cfd0d5 (pale-slate) — never green.

10 · Navigation

Page navigation tabs (canonical)

The canonical page-level tab bar (Workspace, Activity/Org profiles, Admin, Library, Data Clinic — 10+ pages). Track: p-1 h-auto bg-background gap-1 border flex flex-wrap mb-6 — white, hairline-bordered, wraps. Trigger: rounded-md px-3 py-1.5 text-body font-medium flex items-center gap-2 with a leading h-4 w-4 icon; active = muted gray pill (data-[state=active]:bg-muted data-[state=active]:text-foreground data-[state=active]:shadow-sm). Counts (e.g. unread notifications) use a destructive Badge: h-5 min-w-5 px-1 text-helper, capped at "99+".

Nested sub-tabs (secondary contexts only)

The ui/tabs.tsx default (muted track, raised white active pill) is the inverse of the page bar — reserve it for small nested toggles inside a tab's content (e.g. Activities/Organisations inside My Portfolio). Never use it for page-level navigation. Content fades in 150ms (tab-enter).

Breadcrumbs

text-body text-muted-foreground gap-1.5 mb-4; current page font-medium text-foreground; items truncate at max-w-[200px]; separator chevron at 50% opacity.

Pagination

Quiet text buttons: px-2 py-1 rounded text-body text-muted-foreground hover:bg-muted; active page bg-muted text-foreground font-semibold. No bordered button-pagers.

15 · Chart colors (standardized)

Financial-series colors are a single source of truth — src/lib/chart-colors.ts. Every chart, legend, table dot, and tooltip resolves through getTransactionTypeColor(code) / getFinancialSeriesColor(name). Colors are deterministic per IATI code (never assigned by series index), so a Disbursement is the same red on every page.

Supporting palettes

Ranked slate ramp (above): for share-of-whole / Top-10 charts where the only signal is "darker = higher" (sector pies, rankings). Economist extended palette (20 hues anchored on the same red/blue/yellow): only for high-cardinality categorical breakdowns like ~20 DAC sectors. Brand 5-color palette: only for ad-hoc non-financial categories.

RuleWhy
DO resolve every financial series via getTransactionTypeColor / getFinancialSeriesColor — Budget, Planned Disbursements, Commitments, Disbursements, Expenditure all includedA "Disbursement" bar and a type-3 donut slice must be the same red on every page
DON'T hardcode hex, use recharts defaults, or assign colors by series indexIndex-based color flips when series are added/removed
DO use OTHERS_COLOR for unknown codes and "All Others" bucketsDirty data degrades to neutral slate instead of stealing a semantic hue
DO give every chart card the ƒ formula-tooltip button explaining its calculationEstablished convention on analytics cards

Executive Summary

Automated UI audit of the Aether DFMIS platform across 22+ pages. The platform is visually cohesive with a clean dark sidebar, consistent typography, and well-structured navigation. However, several data integrity issues, loading performance concerns, and UI inconsistencies were identified across modules. The most impactful findings relate to missing/unknown data in key financial views, exposed implementation details in the Data Clinic, and percentage calculations exceeding 100% in financial summaries.

Critical Issues

Critical Data Clinic exposes raw SQL commands to users
/data-clinic
The Data Clinic page shows a "Database Migration Required" warning with a raw psql command visible to all users: psql -h your-host -U your-user -d your-db -f frontend/sql/add_data_clinic_fields.sql. This exposes internal implementation details and could confuse non-technical users. The migration prompt should be shown only to administrators and should not display raw shell commands.
Critical Transactions page: all Provider and Receiver show "Unknown"
/transactions
Every transaction in the Transactions list shows Unknown → Unknown for Provider and Receiver columns. This is in contrast to the Planned Disbursements page which correctly shows organization names (e.g. "EU → UNDP", "DFAT → MOPFI"). This suggests transaction provider/receiver data is not being resolved or joined properly.
Critical Pooled Fund shows 4193% utilisation and negative balance
/funds
The "Women's Economic Empowerment through Microfinance and Skills Training" fund shows 4193% utilised with a balance of $-13.5M in red. The progress bar extends far beyond its container. This indicates either a data entry error or a calculation bug where disbursements vastly exceed contributions. The UI should cap the progress bar at 100% and flag anomalies rather than rendering them as normal.
Critical Activity detail shows 250% of committed funds spent
/activities/[id]
An activity detail page shows "Percentage of Committed Funds Spent: 250%" with disbursements ($8M) and expenditures ($7M) totalling $15M against $6M committed. While this may be valid data, the UI presents it without any warning or visual indicator that this is anomalous. The progress bar overflows. Consider capping the visual at 100% and adding an alert badge.

High Issues

High Portfolios page appears stuck on skeleton loading
/partners
The Portfolios page never fully loaded during testing — it remained stuck on skeleton placeholders with no page heading, no data, and no error message. The accessibility tree showed an element marked as busy with an aria-label "Loading partner funding summary". This page may have a failing API call or an infinite loading state with no timeout/error handling.
High All activities show "Reported by: Unknown"
/activities
In the Activity List, every activity displays "Unknown" in the "Reported by" column. Since the Planned Disbursements and Budgets pages correctly show reporting organizations, this appears to be a data join issue specific to the Activities list query rather than missing data.
High All activities show $0 for Total Budgeted and Total Planned Disbursements
/activities
Every activity in the Activity List shows "USD 0" for both "Total Budgeted" and "Total Planned Disbursements" columns. However, the Budgets page shows real budget data (e.g. EUR 4,700,000, AUD 8,500,000). This suggests the Activity List is not aggregating or joining budget data correctly.
High Transactions page: all Finance Type columns show "—"
/transactions
Every transaction displays an em dash "—" in the Finance Type column, suggesting this field is never populated. Since finance type is an important IATI classification, this may indicate missing data or a display bug.
High Slow page load times — navigation timeouts on multiple pages
Multiple pages
Several pages exceeded 20-second navigation timeouts: Dashboard, Activities, Transactions, Analytics Dashboard, and the Activity Detail page. During loading, a full-page spinner with "Loading..." text is shown with no sidebar or navigation — the user loses all context and cannot navigate away. Consider showing the sidebar/shell immediately and loading content progressively.

/organizations
Every organization card displays "Activities: 0" and "Total Budgeted: $0", even for organizations that clearly have associated activities (e.g. World Bank, USAID, EU). This is likely a data aggregation issue in the organizations query.
Medium Policy Markers: IATI badge count contradicts activity count
/policy-markers
Environmental policy marker cards show IATI badge numbers (5, 8, 7, 6) that differ from the activity counts below them (all showing "0 activities"). The IATI badge appears to show the IATI code number, not an activity count — but its position and styling make it look like a count badge, creating confusion.
Medium Library sidebar count (18) doesn't match page count (7)
/library
The sidebar navigation shows "Library 18" but the Document Library page displays "7 documents found". This may be due to the "All Documents" tab filtering differently from the total, or a caching issue with the sidebar count.

SDG 1 (No Poverty) shows 6 active activities with $322.1M committed, but "Organizations: 0 donors", "Top Donors: No donor data", and "Geographic Spread: No geo data". The organization resolution appears to not be working for SDG profile pages.

The Transparency Index page shows "Average Score: 0.0 / 100", "Projects Assessed: 0", "In Good Standing: 0.0%". The score distribution chart is empty. This feature appears to not be computing scores, possibly dependent on the Data Clinic migration that hasn't been run.

Medium Sectors page: some sectors show blank funding column

Organizations list includes obvious test entries: "Agency A (NOA-A)" with ID AA-AAA-123456789 and "Agency B (NOA-B)" with ID BB-BBB-123456789. These should be removed or flagged as test data in a production-like environment.
Low Activity detail: "Actual End: —" uses em dash for empty value
/activities/[id]
The activity detail page shows "Actual End: —" using an em dash for null dates. Other pages use different empty state patterns (blank, "N/A", or "Not set"). Consider standardizing the empty state representation across the application.
Low Dashboard hero cards show inconsistent "by you" vs "your organisation" labels
/dashboard
Dashboard hero stat cards mix labeling: "Financial Transactions" shows "64 Your Organisation / 0 By you" while "Budgets" shows "14 Your Organisation / 0 By you". The terms "By you" and "Your Organisation" are used but the relationship between them isn't clearly explained. For a user with 0 personal contributions across all categories, consider whether the "By you" column adds value.
Low Policy Markers: some cards missing description text
/policy-markers
In the Environmental category, the first card ("Aid Targeting the Objectives of the Convention on Biological Diversity") has no description below it, while neighboring cards do. This creates uneven card heights and visual inconsistency in the grid.

What Works Well

Positive Consistent and professional visual design
The application has a cohesive design language: dark sidebar with clear section groupings, consistent typography, well-spaced cards and tables, and a professional color palette. Page headers follow a uniform pattern with icon + title + subtitle.
Positive Rich feature set across modules
The platform covers an impressive breadth: DFMIS with dashboards, analytics, and activity management; Project Bank with multi-phase review boards; Land Bank with geospatial data; Document Library; Calendar; Working Groups; FAQ system; and multiple profile pages (SDGs, Sectors, Locations, Policy Markers). The navigation is well-organized into logical sections.
Positive Search functionality works well
The global Search page returns results quickly with highlighted matches, grouping results by type (Activities, Organisations). The search box UX is clean and responsive.
Positive Atlas map is interactive and well-integrated
The Atlas/Map view uses MapLibre GL with proper controls (3D toggle, zoom, layer selection, style switching), filters for status/organization/sector, and correctly plots activity locations across Myanmar.
Positive Financial pages (Budgets, Planned Disbursements) show proper data
Unlike Activities and Transactions, the Budgets and Planned Disbursements pages correctly display financial data with proper currency formatting, organization flags/logos, and complete Provider → Receiver chains. These serve as a model for fixing the broken pages.

5. Card Border, Radius & Shadow

Mostly Consistent — minor radius mixing Cards use rounded-xl (12px) but some inner surfaces use 8px or 6px

The standard Card component consistently uses rounded-xl (12px), border-border/60 (0.667px), and a custom multi-layer shadow. However, some pages mix rounded-lg (8px) elements within card layouts (e.g. SDG detail chart containers), and filter/input elements use rounded-md (6px). The 12px for cards and 6px for inputs is a reasonable system — the issue is when 8px appears as an intermediate value with no clear semantic meaning.

4px (codes) → 6px (inputs) → 12px (cards/panels) → 9999px (pills/avatars)

Drop the 8px tier. Promote inner chart containers to rounded-xl (12px) to match their parent card.

8. Status Badges & Color Tokens

Mostly Consistent Status badges follow a clear pattern

Status badges like "Active", "Published", "Implementation" consistently use 12px / weight 600 / 6px radius / padding 2px 10px. Green badges ("Active", "Published") use bg: rgb(220,252,231) with green text. The Badge component in badge.tsx defines proper variants.

Partial Issue Some status colors use hardcoded Tailwind utilities instead of semantic tokens

The codebase has 88+ instances of hardcoded Tailwind color classes like bg-blue-50, bg-green-50, bg-red-50 etc. The badge component also uses hardcoded hex values (#4c5568, #dc2625) for Project Bank pipeline statuses instead of referencing CSS variables. While the visual result is consistent, these hardcoded values won't respond to theme changes and are harder to maintain.

What: Create a shared formatCurrency() utility with two modes: summary ($18.0M) and detail (USD 4,700,000). Fix negative value display from $-13.5M to -$13.5M. Apply consistently across dashboard cards, table cells, chart tooltips.

Priority 3 — Design system improvements

What: Audit all rounded-lg (8px) usage. Promote to rounded-xl (12px) for card-like containers, or demote to rounded-md (6px) for input-like elements. Keep a clean 4-tier scale: 4px → 6px → 12px → full.

P8 Move hardcoded colors to CSS variables

What: Replace the 88+ direct Tailwind color class usages (bg-blue-50, bg-green-100) with semantic CSS variables like --success-bg, --warning-bg, --info-bg. Also move hardcoded hex values in badge.tsx to CSS variables. This enables future theming and ensures a single source of truth for semantic colors.
